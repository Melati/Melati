Persistent Object Engine for Melati.

POEM establishes a lazily instantiated, cached persistence layer over any JDBC compliant SQL database which supports Transactions.

Melati / POEM separation

Guided by the JDepend Report the aim is to separate POEM from Melati in such a way that POEM can be a compiled as a separate source tree, into a separate jar and as a Maven sub project.

The classes in org.melati.util, the following disentangling to be done:

Transaction utils
Move from org.melati.util to org.melati.poem.transaction
MelatiLocale
Move to org.melati.poem.PoemLocale
*Utils
Move to org.melati.poem.utils
*Enumeration
Move to org.melati.poem.utils
CSVParser
Move to org.melati.poem.csv
Interfaces
Move to org.melati.poem
The classes in org.apache should be rewritten and moved to org.melati.poem.transaction
